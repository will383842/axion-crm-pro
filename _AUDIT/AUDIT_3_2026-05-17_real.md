## Bugs CRITIQUES découverts (auraient cassé en prod)

### C1 — User + Workspace models : keyType non configuré pour UUID
`migration 000002` crée `users.id UUID PRIMARY KEY` et `workspaces.id UUID`, mais les Eloquent models n'ont pas :
```php
protected $keyType = 'string';
public $incrementing = false;
```
→ Sans ça, Eloquent croit que `id` est `bigint`, génère mauvais binding, casse Auth::login + relations.

### C2 — `WaterfallOrchestrator.step2_annuaire` upsert contacts avec colonne GENERATED
La migration définit `contacts.normalized_hash` comme `GENERATED ALWAYS AS (...) STORED`.
PostgreSQL refuse tout INSERT/UPDATE sur cette colonne. Le `DB::table('contacts')->upsert([...], ['workspace_id', 'normalized_hash'], ['updated_at'])` va échouer car Laravel envoie tous les fillables incluant la composition de normalized_hash.

### C3 — `WaterfallOrchestrator.step4_dispatch_node_scrapes` : syntaxe `\Queue::push()` invalide
Laravel 12 n'a pas `\Queue::push(new Job, '', queue)`. Bonne syntaxe : `DispatchScrapeJob::dispatch(...)` ou `dispatch(new DispatchScrapeJob(...))`.

### C4 — SsrfGuard défini mais **jamais appelé** dans HttpInseeClient/Annuaire/BAN/FranceTravail
Le guard SSRF est code mort. Tous les `Http::get(...)` partent sans validation.

### C5 — Sanctum migration `personal_access_tokens.tokenable_id UUID` vs default Sanctum BIGINT
Si Sanctum publie sa propre migration, conflit. Si on garde la mienne, Sanctum cast en `int` côté code → erreur.

### C6 — `ClassifierService.autoTag` : `array_merge` sur `$company->signals` potentiellement null
```php
$signals = $company->signals ?: [];   // OK dans certaines branches
$signals['classification'] = array_merge($signals['classification'] ?? [], ...);
```
Si premier appel sans cast 'array' → erreur.

### C7 — `EnsureFrontendRequestsAreStateful` doublé
`bootstrap/app.php` : `$middleware->statefulApi();` + `$middleware->web(append: [EnsureFrontendRequestsAreStateful::class])`. Double-bind cause double-execution.

### C8 — composer.json n'a pas `spatie/laravel-query-builder`
J'utilise `QueryBuilder::for(Company::query())` dans CompaniesController mais la dépendance Spatie est dans `composer.json` v6.2 ✓ — vérifié, OK.

### C9 — `migration 000003 companies.geo_point` PostGIS — pas de `GeomFromText`
Le ST_MakePoint en migration `WaterfallOrchestrator.step8_geocode` utilise `ST_SetSRID(ST_MakePoint(?, ?), 4326)`. Bon. Mais le type colonne est `geometry(Point, 4326)` ✓. OK.

### C10 — Frontend routeTree : `import { RootLayout }` from './RootLayout' alors qu'il faut le path local
Vérifié : OK le fichier est dans le même dossier `frontend/src/app/`.
