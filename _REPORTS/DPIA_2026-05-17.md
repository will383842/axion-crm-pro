## 1. Description du traitement

### 1.1 Finalités
- Identifier des entreprises B2B françaises susceptibles d'être intéressées par les
  prestations Axion-IA (audit IA, mission PME/ETI, grand programme)
- Constituer un répertoire de décideurs (dirigeant légal, C-level) à contacter par
  cold-email + LinkedIn outreach (Phase 2)

### 1.2 Base légale (RGPD art. 6)
- **Intérêt légitime art. 6.1.f** — B2B uniquement, équilibre intérêts/droits du sujet
  (dirigeants en fonction officielle, données publiques INSEE/BODACC), avec opt-out
  immédiat et cross-workspace
- **Pas de consentement** requis car contexte B2B + données publiques uniquement

### 1.3 Catégories de données
- **Identité légale** : raison sociale, SIREN/SIRET, forme juridique, NAF, effectif INSEE
  (catégorie publique INSEE)
- **Coordonnées entreprise** : adresse postale, site web, téléphone standard, LinkedIn corporate
- **Dirigeants légaux** : nom, prénom, rôle (gérant/PDG/président) — publié au RNCS via
  annuaire-entreprises.data.gouv.fr (donnée publique légale)
- **C-level (ETI/Grandes)** : nom, titre, LinkedIn — issus de pages corporate publiques +
  communiqués de presse
- **Email professionnel** : généré par patterns + validation SMTP cascade (jamais collecté
  depuis Phantombuster/scraping LinkedIn personnel)
- **Signaux business** : levées de fonds, recrutements actifs (France Travail), BODACC

### 1.4 Catégories EXCLUES
- Aucune donnée sensible (origine, religion, santé, opinion politique, orientation sexuelle…)
- Aucune donnée biométrique
- Aucune donnée mineur
- Aucun email personnel (gmail/hotmail) — uniquement emails domaine entreprise

### 1.5 Durées de conservation
| Donnée | Conservation |
|--------|--------------|
| Audit logs (compliance) | 24 mois puis archivage chiffré |
| Scraping payload (PII brut) | 90 jours (auto-purge) |
| Email validations cache | 30 jours (TTL) |
| IPs (audit + sessions) | 30 jours puis anonymisées /24 (IPv4) ou /48 (IPv6) |
| Contacts non contactés | indéfini (mais purge sur demande RGPD ≤ 30j) |
| Opt-out cross-workspace | indéfini (cf. art. 17) |

## 2. Nécessité et proportionnalité

- **Données collectées proportionnées** : uniquement ce qui sert à qualifier la cible
  Axion-IA (taille, secteur NAF, signaux IA-readiness)
- **Pas de profilage discriminatoire** : classification automatique LLM produit un score
  d'opportunité commerciale, jamais une décision impactant la personne physique. Décision
  finale = opérateur humain Axion-IA
- **Minimisation appliquée** : pas de date naissance, pas d'adresse personnelle, pas de
  numéro mobile personnel
- **Anonymisation par défaut** : IPs sessions/audit anonymisées > 30j

## 3. Mesures de sécurité (RGPD art. 32)

### 3.1 Techniques
- **Chiffrement au repos** : Postgres TDE Hetzner + S3 backups AES-256
- **Chiffrement en transit** : TLS 1.3 forced, HSTS preload 12 mois, COOP/CORP
- **Authentification forte** : 2FA TOTP obligatoire (`EnforceFirstLoginSetup` middleware)
- **Audit hash chain** : SHA-256 append-only vérifiable, détection tampering automatique
- **RLS PostgreSQL** : isolation rangée par workspace_id sur 30+ tables
- **SSRF guard** : refus IPs privées + metadata cloud (back + workers)
- **CSP strict prod** : pas de unsafe-inline, COEP credentialless
- **Sanitize prompt-injection** : variables `ext_*` filtrées avant envoi LLM
- **Rate limiting** : login 5/min, magic-link 3/IP/10min
- **Brute force protection** : lock 24h après 10 fails

### 3.2 Organisationnelles
- **RBAC 4 rôles** : owner/admin/operator/viewer (Spatie Permission teams)
- **Audit log immuable** : toute mutation enregistrée + verifyChain quotidien
- **Logs centralisés** : Loki retention 30j + Tempo traces
- **Alertes** : Alertmanager → Slack #axion-crm-alerts (warning) + Telegram oncall (critical)
- **Backups** : pgbackrest hourly Hetzner OBS + réplication Backblaze B2 (3-2-1)
- **DR drill** : trimestriel, RPO 1h / RTO 4h documenté

## 4. Droits des personnes (RGPD art. 15-22)

| Droit | Implémentation |
|-------|----------------|
| Accès (art. 15) | `POST /api/v1/rgpd/requests {type:access, subject_email}` |
| Rectification (art. 16) | `POST /api/v1/rgpd/requests {type:rectification}` |

| Décision automatisée (art. 22) | **Non applicable** : profilage indirect uniquement, décision finale humaine |

## 6. Évaluation des risques résiduels

| Risque | Probabilité | Gravité | Mesure | Risque résiduel |
|--------|-------------|---------|--------|-----------------|
| Fuite SIREN/dénomination (data publiques) | Élevée | Faible | RLS + audit logs | **Faible** |
| Fuite emails professionnels | Moyenne | Moyenne | Chiffrement repos + opt-out global + transaction atomique erasure | **Faible** |
| Fuite signaux business sensibles (CA bilans publics) | Moyenne | Faible | Données déjà publiques (BODACC, AnnuaireEntreprises) | **Très faible** |
| Compromission credentials provider LLM | Faible | Moyenne | Doppler + rotation 180j + cost cap workspace | **Faible** |
| Attaque SSRF via website scraper | Faible | Élevée | SsrfGuard.ts + SsrfGuard.php double-couche | **Faible** |
| Brute force credentials owner | Moyenne | Élevée | 2FA TOTP + lock 24h + throttle 5/min + HIBP check | **Très faible** |
| Tampering audit logs | Très faible | Élevée | Hash chain SHA-256 + verifyChain quotidien | **Très faible** |
| Profilage discriminatoire algo | Faible | Moyenne | Décision humaine finale + AI Act register limited | **Faible** |
